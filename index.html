✍️ 或貼上純文字辨識
把檢驗數值文字貼上（例如：血壓 130/85、空腹血糖 98、LDL 138、HbA1c 5.9），用 AI 辨識成結構化數值。
🏥 健康存摺（My Health Bank）
步驟：健保快易通 App 或 健康存摺網站 → 登入 → 下載／匯出（FHIR 或 JSON）→ 於此上傳。
自動擷取：血壓、空腹血糖、糖化血色素、總膽固醇、HDL、體重、身高、心率等檢驗值，並自動解析用藥紀錄（處方／調劑）整理到「💊 用藥紀錄」。
支援 FHIR／JSON、.zip／.gz 壓縮檔（本機解析）；若上傳的是 PDF 報告，會改用 AI 視覺辨識（需 AI 金鑰，報告會傳送至 AI 服務判讀）。
✍️ 或貼上文字／JSON 辨識
貼上健康存摺的 FHIR／JSON 文字（本機解析，不外傳）；或直接貼一般檢驗數值文字（用 AI 辨識）。

🔥 Firebase 雲端同步 （僅擁有者）
✅ 已內建 Firebase 設定，直接輸入 Email／密碼登入或註冊即可，跨裝置自動同步健康紀錄、檢驗數據與個人檔案。首次請按「🆕 註冊新帳號」，之後其他裝置用同帳密登入即同步。
⚙️ 進階：自訂 Firebase 設定（一般免填）
若要改用你自己的另一個 Firebase 專案，貼上其 firebaseConfig；留空則使用內建設定。
❓ 如何取得 Firebase 設定？（手機可完成）
全程在手機瀏覽器即可操作，不需電腦。只需做一次；之後其他裝置用同一組 Email/密碼登入就會同步。
  1. 到 console.firebase.google.com 用 Google 帳號登入 →「建立專案」（名稱隨意，Analytics 可略過）。
  2. Firestore：左側「建構 → Firestore Database」→「建立資料庫」→ 地區選 asia-east1（近台灣）→ 先選「測試模式」即可（規則稍後改）。
  3. 登入方式：「建構 → Authentication → 開始使用」→ 選「電子郵件/密碼」→ 啟用 → 儲存。
  4. 拿設定：齒輪「專案設定」→ 下方「你的應用程式」→ 點 </> 網頁圖示 → 註冊 App → 複製整段 const firebaseConfig = {…} 貼到上面欄位。
  5. 設定規則：回 Firestore →「規則」分頁 → 貼上下方規則 → 發布。
  6. 回到這裡輸入 Email／密碼 → 按「🆕 註冊新帳號」→ 完成。其他手機/電腦用同帳密「🔑 登入」即同步。
Firestore 安全規則（只允許本人讀寫自己 users/{uid} 底下的資料；健康資料存於 users/{uid}/lifespan/data）： 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}
若你的專案已有 match /users/{userId}/… 這類規則（例如與其他 App 共用），通常不需更動即可支援本 App。
💡 設定可整段貼（含 const firebaseConfig =），系統會自動解析。若之後換了 Firebase 設定，請重整頁面再登入。密碼只用於登入、不會被上傳或儲存到雲端。
⚙️ 進階：改用 Google Apps Script（需電腦部署）
此為替代方案，適合想用 Google 試算表當資料庫、且能存取電腦的人。手機無法建立 Apps Script，部署步驟需在電腦上完成。設定後可額外使用每日/每週自動 Email 健康守護。
  1. 開一份 Google 試算表 →「擴充功能 → Apps Script」，貼上 health-backend.gs 全部內容並存檔。
  2. 函式選 setupSheets → 執行 → 授權（進階 → 前往 → 允許）。
  3. 「部署 → 新增部署作業 → 網頁應用程式」；執行身分＝我、存取權＝所有人。
  4. 複製結尾為 /exec 的網址貼上，按「🔌 測試連線」。

ℹ️ 關於
LifeSpan 健康預測 是一套個人健康管理與延壽輔助系統。
預測模型整合了 BMI、腰高比、血壓、代謝、運動、睡眠、飲食、吸菸等實證流行病學因子，估算生理年齡、預期壽命與慢性病相對風險。

本工具僅供衛教與自我管理參考，不構成醫療建議。· 2026
