# 🫀 LifeSpan 健康預測

## 個人健康管理與延壽系統

輸入健康數據 → 預測生理年齡與預期壽命 → 慢性病風險分級 → 主動預警與個人化延壽建議。

**線上使用：** 開啟 `index.html`，或部署至 GitHub Pages（`https://cyysongyy.github.io/health/`）。

> ⚠️ **免責聲明：** 本系統為健康自我管理與衛教用途，所有預測皆為基於公開流行病學研究的統計估算，**不能取代醫療診斷**。如有健康疑慮，請諮詢合格醫師。危急症狀請立即就醫。

> 本系統與「補救教學系統」為兩個完全獨立的專案，彼此無關、不共用資料。

## 🦶 足部訓練（主程式分頁）

在主程式導覽列的 **🦶 足部訓練** 分頁中，內建走路前暖身與垂足背屈復健兩個模組，與健康預測資料互不相依：

| 模組 | 內容 | 時長 |
|---|---|---|
| 🔥 足部暖身 | 足底放鬆 → 小腿後側伸展 → 腳踝喚醒 → 臀部啟動 | 約 4 分鐘 |
| 💪 垂足背屈復健 | 毛巾伸展、牆面弓步、坐姿背屈、阻力帶背屈、離心提踵（右腳患側） | 每日 5 動作 |

- **語音導引（zh-TW）**：每個步驟自動播報動作提示（Web Speech API）
- **自動計時與換動作**：倒數計時 + 自動進入下一步／下一組
- **可調參數並記憶**：組數與維持秒數可增減，透過 `localStorage` 記住設定
- **示意圖內嵌**：以 SVG 繪製（綠＝右腳患側、藍＝左腳健側、橘＝動作重點、紅＝拉伸感），無外部圖片相依
- **不干擾主程式**：樣式以 `#foot-app` 作用域隔離、程式以 `FT` 命名空間封裝，與健康預測邏輯零衝突

> ⚠️ 本模組為衛教與自主訓練用途，不能取代醫療診斷。若疼痛加劇、麻木或不適，請諮詢復健科或物理治療師。

## 🔗 資料串接（匯入）

系統可自動解析並帶入外部健康資料，全程在瀏覽器端完成、資料不外傳：

| 來源 | 檔案格式 | 自動擷取 |
|---|---|---|
| 🏥 健康存摺（My Health Bank） | FHIR Bundle / JSON | 血壓、血糖、糖化血色素、總膽固醇、HDL、體重、身高、心率（依 LOINC 代碼） |
| ⌚ 穿戴裝置 | Apple 健康 `export.xml` / Google Fit JSON | 每日步數→每週運動量、靜止心率、睡眠時數 |
| 📎 現有 App / 附件 | JSON / CSV | 依欄位名（含中英文）自動對應 |

> **關於健保：** 台灣全民健保基於資安設計，未開放第三方 App 直接讀取。正確流程是由本人到「健保快易通／健康存摺」匯出自己的資料後於此匯入。

## 🔥 Firebase 雲端同步（推薦，手機即可設定）

前端直接串接 Firebase（Firestore + Email/密碼登入），**免部署伺服器**，設定全程可在手機瀏覽器完成：

1. 到 [Firebase 主控台](https://console.firebase.google.com/) 建立專案。
2. 開啟 **Firestore Database**（地區建議 `asia-east1`）與 **Authentication → 電子郵件/密碼**。
3. 專案設定 → 新增 Web App → 複製整段 `firebaseConfig`。
4. 於 App「設定 → 🔥 Firebase 雲端同步」輸入 Email/密碼 → 註冊 → 登入即可（App 已內建預設 Firebase 設定；若要改用自己的專案，於「進階」貼上 config）。其他裝置用同帳密登入即同步。
5. 於 Firestore「規則」貼上（只允許本人讀寫自己的資料）：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

> 健康資料存於 `users/{uid}/lifespan/data`，只有登入本人能讀寫。若專案已有 `match /users/{userId}/…` 規則（與其他 App 共用），通常無需更動。密碼僅用於登入，不會被上傳或儲存到雲端。

## ☁️ 雲端後台與自動化（進階，`health-backend.gs`）

以 Google Apps Script + 試算表作為個人健康資料的雲端中心：儲存、比對、跨裝置同步，並可自動化。**部署步驟需在電腦上完成（手機無法建立 Apps Script）**，額外提供每日/每週自動 Email 守護。

- `setupWeeklyTrigger()`：每週一早上 8 點自動產生健康週報並寄 Email。
- `setupDailyWatch()`：每日早上 9 點自動執行警示引擎，偵測到危險門檻或惡化趨勢才主動寄警告信（無警示則不打擾）。
- `setAiConfig('gemini','你的KEY')`：金鑰存於 Script Properties，設定後每日守護信與週報會附上 AI 個人化解讀。

設定：開啟 Google 試算表 → 擴充功能 → Apps Script → 貼上 `health-backend.gs` → 執行 `setup*` 函式授權 → 部署為網頁應用程式（執行身分：我；存取：所有人）→ 複製網址貼入 App「設定 → 雲端後台」→ 測試連線。

## 💾 資料儲存與隱私

- **本機模式**：所有資料存於瀏覽器 `localStorage`，不外傳。
- **雲端模式**：設定 GAS 後台後同步至你自己的 Google 試算表，跨裝置查看。
- **隱私**：API Key 僅存於你的瀏覽器或 Script Properties，直接呼叫 API，不經任何第三方伺服器。
